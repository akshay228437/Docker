# Q1 – Concept Guide: Container Cleanup & Image Removal

## 🎯 Goal

Understand how to properly **decommission a Docker service** by:

* Stopping a running container
* Removing the container instance
* Cleaning up unused images

---

## 🛑 1. Stop the Container

### Key Concept

* Containers must be **stopped before removal** (best practice)
* Stopping ensures the application shuts down gracefully

### What to Know

* Docker tracks running containers separately from stopped ones
* Stopping affects only runtime execution, not disk presence

---

## 🗑️ 2. Remove the Container

### Key Concept

* Removing a container deletes its **runtime instance**
* After removal, it will no longer appear in container listings

### Why Order Matters

* A running container cannot be safely removed without stopping it first
* Proper workflow ensures **clean lifecycle management**

---

## 🧹 3. Remove the Image

### Key Concept: Images vs Containers

* **Container** = running instance
* **Image** = blueprint used to create containers

### When to Remove an Image

* If no containers depend on it
* To free up disk space
* To keep system clean in development environments

---

## 🔁 Lifecycle Flow

```
Container Running → Stop Container → Remove Container → Remove Image
```

---

## ⚠️ Common Mistakes

* Trying to remove a running container directly
* Forgetting that images can still exist after container deletion
* Removing images still used by other containers

---

## 🧠 Key Takeaway

Docker cleanup follows a strict hierarchy:

* Stop first
* Remove container second
* Remove image last

> Proper cleanup prevents resource leaks and keeps environments stable.
