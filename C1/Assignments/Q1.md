# Q1 – Concept Guide: Pulling a Specific Docker Image

## 🎯 Goal

Learn how to:

* Pull a **specific version** of an image
* Ensure the correct artifact is retrieved for deployment

---

## 📦 Image Retrieval

### Key Concept: Image Name + Tag

* Docker images follow:

  ```
  <image>:<tag>
  ```
* The `tag` defines the **exact version**.
* If no tag is provided, Docker uses `latest` by default.

### Why Tag Matters

* Ensures **consistency across environments**
* Prevents issues caused by unexpected version changes

---

## 📥 Pulling from Registry

### Key Concept: Default Registry

* Docker pulls images from the official registry (Docker Hub) unless specified otherwise
* Official images (like nginx) are maintained and trusted sources

---

## 🧾 Logging (Concept Only)

* Docker pull generates logs showing:

  * Layers being downloaded
  * Status messages (e.g., success/failure)
* These logs can be captured for **audit and verification**

---

## ✅ Expected Understanding

After completing:

* You should have the **correct image version locally**
* You should know how to confirm:

  * The image name
  * The image tag

---

## ⚠️ Common Mistakes

* Not specifying a tag (pulls `latest`)
* Using incorrect image name or casing
* Assuming all versions behave the same

---

> Focus: Always use **explicit image versions** in production workflows.
