# Q3 – Concept Guide: Background Nginx Container

## 🎯 Goal

Run and manage a web service container in background mode and observe its lifecycle.

---

## 🚀 1. Run in Background

### 💡 Hint

* Use Docker’s **detached mode flag**
* Combine it with **interactive TTY option if needed for practice**
* Think: *run container without attaching terminal*

👉 Hint command idea:

```
use -d or -dt style flags
```

---

## 🌐 2. Web Service Behavior

### 💡 Hint

* Nginx runs continuously (no shell interaction required)
* It should keep running after terminal closes

👉 Think: *service container vs interactive container*

---

## 🔍 3. Check Running State

### 💡 Hint

* There is a Docker command to list **only active containers**
* Useful to confirm your service is running in background

👉 Think: *"what is currently running?"*

---

## 📜 4. View Logs

### 💡 Hint

* Docker allows inspection of container output
* Look for startup confirmation messages from Nginx

👉 Hint command idea:

```
logs <container-name>
```

---

## 🔁 5. Lifecycle Control

### 💡 Hint

* Containers can be stopped and restarted without re-creating them
* Use separate commands for:

  * stop
  * start

👉 Think: *state change of same container*

---

## 🧠 Key Idea

> Run service in background → verify it’s running → inspect logs → manage lifecycle

---

## ⚠️ Common Mistakes

* Forgetting to detach container (it blocks terminal)
* Confusing image run vs container state
* Not checking logs after startup

---

> Focus: background execution (-d/-dt), verification, logs, and restart flow
